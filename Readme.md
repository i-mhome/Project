# 전지환 프로젝트 제안서

## 1. 프로젝트 이름

로보 자비스(회계 정산 ver)

---

## 2. 한 문장 설명

동아리 행사 참여 명단과 체크 명단을 직접 입력하거나 이미지 OCR로 읽어 미입금자를 찾고, 1인당 정산 금액과 정산 결과 CSV 파일을 자동으로 만들어주는 프로그램입니다.

---

## 3. 풀려는 문제 또는 만들려는 이유

저는 현재 로보라는 과동아리 회장으로 활동하고 있습니다. 동아리에서는 행사나 회식이 있을 때마다 참여자 명단을 확인하고, 전체 금액을 인원 수로 나눈 뒤, 누가 입금했는지 다시 비교해야 합니다. 기존에는 엑셀이나 카카오톡 체크 명단을 보면서 직접 비교했기 때문에 인원이 많아질수록 미입금자 확인이 번거로웠습니다. 그래서 참여 명단과 체크 명단을 입력하거나 캡쳐 이미지에서 읽어와 자동으로 미입금자를 찾고, 1인당 금액과 결과표를 만들어주는 정산 보조 프로그램을 만들고자 합니다.

---

## 4. 핵심 기능 및 우선순위

### [MUST] 반드시 구현할 기능

1. **직접 입력 방식 정산 기능**
   - 전체 명단과 체크된 사람 명단을 직접 입력받아 리스트로 변환합니다.
   - 전체 명단에는 있지만 체크 명단에는 없는 사람을 미입금자로 분류합니다.

2. **1인당 정산 금액 계산 기능**
   - 행사 총 금액을 입력받고 전체 인원 수로 나누어 1인당 납부 금액을 계산합니다.
   - 숫자가 아닌 값이 입력되면 `try-except`로 오류를 처리하고 다시 입력받습니다.

3. **정산 결과 출력 기능**
   - 전체 인원, 체크된 사람, 미입금자, 전체 인원 수, 1인당 금액을 터미널에 출력합니다.

4. **CSV 저장 기능**
   - 정산 결과를 `settlement_result.csv` 파일로 저장합니다.
   - 이름, 상태, 납부금액을 표 형태로 저장하여 엑셀에서 확인할 수 있게 합니다.

5. **OCR 이미지 정산 기능**
   - 참여 명단 이미지 폴더와 체크 명단 이미지 폴더를 입력받습니다.
   - 각 폴더 안의 이미지 파일을 EasyOCR로 읽어 이름 리스트로 변환합니다.
   - OCR로 읽은 명단을 기존 정산 처리 함수에 연결하여 미입금자를 계산합니다.

### [NICE] 있으면 좋은 기능

1. **전체 명단에 없는 체크 이름 경고 기능**
   - 체크 명단에는 있지만 전체 명단에는 없는 이름을 따로 출력하여 오타나 잘못된 입력을 확인할 수 있게 합니다.

2. **여러 이미지 자동 처리 기능**
   - 이미지 파일명을 하나씩 입력하지 않고, 폴더 안의 `.png`, `.jpg`, `.jpeg` 파일을 자동으로 읽습니다.

3. **중복 이름 제거 기능**
   - 여러 이미지에서 같은 이름이 반복해서 읽히는 경우 한 번만 저장합니다.

### [LATER] 시간이 남으면 추가할 기능

1. **동명이인 구분 기능**
   - 이름만으로 구분이 어려운 경우 학번 일부나 고유 번호를 함께 사용하도록 확장합니다.

2. **OCR 결과 수정 기능**
   - OCR 인식 결과가 틀렸을 때 사용자가 직접 수정할 수 있도록 합니다.

3. **회비 입출금 및 잔액 관리 기능**
   - 행사 정산뿐만 아니라 동아리 회비 입금, 출금, 현재 잔액까지 관리할 수 있도록 확장합니다.

## 5. 예상 사용자

동아리 회장, 총무, 학회장, 소모임 운영자 등 여러 명의 회비와 행사 정산을 관리해야 하는 사람입니다. 저 또한 실제 사용자에 포함됩니다.

## 6. 결과물 형태

### CLI 프로그램

이 프로젝트는 화면 디자인보다 명단 비교, 정산 금액 계산, 미입금자 분류, CSV 저장 기능이 중요하므로 CLI 형태가 적절하다고 판단하였습니다. 사용자는 터미널에서 메뉴를 선택하고, 직접 입력 정산 또는 OCR 이미지 정산 중 하나를 선택하여 사용할 수 있습니다.

## 8. 기대효과

이 프로그램을 사용하면 행사 정산 시 전체 참여자와 입금 완료자를 직접 비교하는 시간을 줄일 수 있습니다. 특히 카카오톡 체크 명단이나 참여 명단 캡쳐본을 OCR로 읽어올 수 있기 때문에, 많은 인원을 하나씩 입력해야 하는 번거로움을 줄일 수 있습니다. 또한 정산 결과를 CSV 파일로 저장하여 엑셀에서도 확인할 수 있어 동아리 회장이나 총무의 반복적인 정산 업무를 줄이는 데 도움이 됩니다.

---

## 9. 한계와 개선 방향

현재 프로그램은 이름 문자열을 기준으로 명단을 비교합니다. 따라서 동명이인이 있는 경우 이름만으로는 사람을 구분하기 어렵습니다. 실제 사용 시에는 이름 뒤에 학번 일부나 번호를 붙여 고유한 식별자로 관리하는 방식이 필요합니다. 또한 OCR 인식률은 이미지 품질, 글자 크기, 파일명 및 경로 형식에 영향을 받을 수 있으므로, 향후 OCR 결과 수정 기능을 추가하면 더 안정적으로 사용할 수 있습니다.
