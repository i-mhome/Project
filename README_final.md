# 동아리 정산 프로그램 실행 절차 최종본

## 1. 프로젝트 소개

이 프로젝트는 동아리 행사 후 전체 참여 명단과 체크 완료 명단을 비교하여 미입금자를 자동으로 확인하는 Python 프로그램입니다.

전체 명단과 체크 명단을 직접 입력할 수 있으며, 이미지 캡처본을 이용한 OCR 명단 인식 기능도 사용할 수 있습니다. 또한 행사 총 금액을 입력하면 1인당 납부 금액을 계산하고, 정산 결과를 CSV 파일로 저장할 수 있습니다.

---

## 2. 실행 준비

프로젝트 최종본 파일을 다운로드한 뒤, 압축을 해제합니다.

압축을 해제한 프로젝트 폴더를 Visual Studio Code에서 엽니다.

프로그램을 실행하기 위해 Python이 설치되어 있어야 합니다.

OCR 기능을 사용하려면 `easyocr` 라이브러리를 추가로 설치해야 합니다.

---

## 3. 라이브러리 설치 방법

Visual Studio Code에서 프로젝트 폴더를 연 뒤, 터미널을 실행합니다.

터미널에서 다음 명령어를 입력합니다.

```text
pip install easyocr
```

만약 위 명령어가 실행되지 않으면 다음 명령어를 사용합니다.

```text
python -m pip install easyocr
```

`csv`와 `os`는 Python 기본 라이브러리이므로 따로 설치하지 않아도 됩니다.

---

## 4. 프로그램 실행 방법

Visual Studio Code에서 프로젝트 폴더를 연 뒤, 실행할 Python 파일을 엽니다.

그 다음 키보드에서 `F5`를 눌러 프로그램을 실행합니다.

만약 `F5` 실행이 되지 않는 경우, Visual Studio Code의 오른쪽 위에 있는 실행 버튼을 눌러도 됩니다.

프로그램이 정상적으로 실행되면 터미널 창에 메뉴가 출력됩니다.

---

## 5. 프로그램 사용 절차

프로그램을 실행하면 다음과 같은 메뉴가 출력됩니다.

```text
===== 로보 자비스 정산 프로그램 =====
1. 직접 입력으로 정산하기
2. OCR 이미지로 정산하기
3. 종료
```

사용자는 원하는 기능 번호를 입력하여 프로그램을 사용할 수 있습니다.

---

## 6. 직접 입력 방식

직접 입력 방식을 선택하면 전체 명단과 체크 완료 명단을 직접 입력합니다.

입력 예시는 다음과 같습니다.

```text
김철수, 이영희, 박민수
```

전체 명단에는 있지만 체크 완료 명단에는 없는 사람은 미입금자로 분류됩니다.

예를 들어 전체 명단이 다음과 같고,

```text
김철수, 이영희, 박민수
```

체크 완료 명단이 다음과 같다면,

```text
김철수, 이영희
```

프로그램은 `박민수`를 미입금자로 판단합니다.

---

## 7. OCR 이미지 방식

OCR 이미지 방식을 사용하려면 실행 파일이 있는 프로젝트 폴더 안에 이미지 폴더를 직접 만들어야 합니다.

예를 들어 전체 참여 명단 캡처본을 넣을 폴더와 체크 완료 명단 캡처본을 넣을 폴더를 각각 만들 수 있습니다.

폴더명은 사용자가 원하는 이름으로 만들 수 있습니다.

예시는 다음과 같습니다.

```text
all_images
paid_images
```

각 폴더의 의미는 다음과 같습니다.

| 폴더명 | 설명 |
|---|---|
| all_images | 전체 참여 명단 캡처본을 넣는 폴더 |
| paid_images | 체크 완료 또는 입금 완료 명단 캡처본을 넣는 폴더 |

프로그램 실행 후 OCR 이미지 방식을 선택하면 각 이미지 폴더명을 입력합니다.

```text
참여 명단 이미지 폴더명을 입력하세요: all_images
체크 명단 이미지 폴더명을 입력하세요: paid_images
```

이때 프로그램에 입력하는 폴더명은 실제로 만든 폴더명과 동일해야 합니다.

예를 들어 폴더명을 `all_images`로 만들었다면 프로그램에도 `all_images`라고 입력해야 합니다.

---

## 8. OCR 이미지 사용 시 주의사항

OCR 기능은 이미지 속 글자를 인식하여 명단을 읽어오는 기능입니다.

하지만 OCR 인식이 아직 완벽하지 않기 때문에 캡처본을 최대한 깔끔하게 준비해야 합니다.

캡처본을 준비할 때는 다음 사항을 지켜야 합니다.

1. 글자가 흐리지 않도록 선명하게 캡처합니다.
2. 이름 또는 닉네임이 잘리지 않도록 캡처합니다.
3. 배경이 복잡하지 않은 화면을 사용하는 것이 좋습니다.
4. 글자 크기가 너무 작지 않도록 캡처합니다.
5. 전체 명단과 체크 명단에서 같은 사람은 반드시 동일한 닉네임을 사용해야 합니다.

예를 들어 전체 명단에는 `홍길동`이라고 되어 있고, 체크 명단에는 `길동`이라고 되어 있으면 프로그램은 서로 다른 사람으로 인식할 수 있습니다.

따라서 전체 명단과 체크 명단의 이름 또는 닉네임은 반드시 동일하게 맞춰야 합니다.

---

## 9. 실행 결과

프로그램 실행 후 다음 정보가 터미널에 출력됩니다.

- 전체 명단
- 체크 완료 명단
- 미입금자 명단
- 전체 인원 수
- 1인당 납부 금액
- 전체 명단에 없는 이름

또한 실행 결과는 CSV 파일로 저장됩니다.

```text
settlement_result.csv
```

CSV 파일에는 이름, 상태, 납부 금액 정보가 저장됩니다.

예시는 다음과 같습니다.

```csv
이름,상태,납부금액
김철수,체크완료,20000
이영희,체크완료,20000
박민수,미입금,20000
```

## 10. 주의사항

1. Python이 설치되어 있어야 합니다.
2. OCR 기능을 사용하려면 easyocr 라이브러리를 설치해야 합니다.
3. OCR 이미지 폴더는 실행 파일이 있는 프로젝트 폴더 안에 만들어야 합니다.
4. 이미지 폴더명은 프로그램에 입력하는 폴더명과 동일해야 합니다.
5. OCR 인식 결과는 이미지 품질에 따라 달라질 수 있습니다.
6. 전체 명단과 체크 명단의 이름 또는 닉네임은 동일하게 입력되어야 합니다.
7. 이름 또는 닉네임을 기준으로 비교하므로 동명이인이나 비슷한 닉네임이 있는 경우 구분이 어려울 수 있습니다.

## 12. 프로젝트를 통해 배운 점

이 프로젝트를 통해 실제 동아리 운영에서 발생하는 정산 문제를 Python 프로그램으로 해결해볼 수 있었습니다.

특히 명단 비교, 금액 계산, CSV 저장 기능을 함수로 나누어 구현하면서 프로그램 구조를 이해할 수 있었습니다.

또한 OCR 기능을 사용하면서 외부 라이브러리를 설치하고 활용하는 방법을 배웠고, 프로그램을 다른 사람이 실행할 수 있도록 실행 절차를 문서로 정리하는 것의 중요성을 알게 되었습니다.
